# 🐶phone-repair-management-system
 
基于芋道（Ruoyi-Vue Pro 芋道微服务版）开发的手机维修门店管理系统，实现维修工单、配件库存、出入库流水、多角色和多门店权限管控全业务流程。

## 一、技术栈
 
后端（芋道微服务基座）
 
Spring Cloud Alibaba、芋道Ruoyi微服务框架、SpringBoot、MyBatis-Plus、MySQL、Redis、Nacos、Gateway等
 
前端
 
Vue3 + Element Plus + TypeScript（芋道前端模板）、Axios、ECharts
 
部署工具
 
Maven、Git

## 二、分角色业务功能介绍
### 1. 系统管理员（最高权限）
1. 账号角色管理：新增/停用店长、维修人员账号，自定义分配角色权限；
2. 基础数据维护：统一录入配件分类、配件基础信息，维护门店全局基础配置；
3. 全量数据查看：可查看所有门店全部维修工单、出入库流水、完整库存数据；
4. 系统管控权限：全局操作日志查询、全量数据导出、库存预警阈值统一配置。

### 2. 门店店长（门店管理权限）
1. 工单管理：查看本店全部维修工单；
2. 库存管控：查看本店完整配件库存、出入库流水，接收库存预警提醒，提交配件采购申请；
3. 人员管理：查看本店维修人员工单完成量，仅可调整本店人员基础信息；
4. 经营看板：查看本店营收、配件消耗、故障分布可视化统计图表。

### 3. 维修人员（一线操作权限，数据隔离）
1. 维修单操作：仅可新建、编辑、查看自身名下维修工单，更新维修进度、完工提交结算；
2. 配件流转：根据维修工单申领维修配件，维修剩余配件执行退库操作；
3. 个人流水查询：仅能查看本人配件领用、退库记录，无法查看其他人员流水；
